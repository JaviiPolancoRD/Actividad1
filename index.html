Tema 1: Elementos de paginas web

I. Origenes de la web

  1. Web 2.0

    2. Los términos web 2.0 y web social aluden a aquellos sitios web que facilitan compartir información, la interoperabilidad, el diseño centrado en el usuario y la colaboración en la World Wide Web. Permiten a los usuarios interactuar y colaborar entre sí, como creadores de contenido.

  2. Web 3.0 3

    3. La Web 3.0 se basa en la inteligencia artificial (IA) para desarrollar ordenadores capaces de comprender el significado o el contexto de las peticiones de los usuarios y responder a solicitudes complejas con mayor rapidez.

  3. La W3C y los estandares

    4. Los estándares Web W3C son protocolos, pautas y tecnologías inter-operativas e internacionales creadas con la finalidad de guiar la Web a un mejor rendimiento y mejorar la funcionalidad operativa para los navegadores y usuarios.

II. Lenguajes de etiqueta

  1. SGML

    2. SGML (lenguaje de marcado generalizado estándar) es un estándar internacional para la definición de lenguajes de marcado ; es decir, es un metalenguaje. El marcado consiste en notaciones llamadas etiquetas que especifican la función de un fragmento de texto o cómo debe mostrarse.

  2. XML

    3. El XML es un formato de archivo muy utilizado en la web para describir y almacenar datos, especialmente para intercambiar datos entre diferentes sistemas.

  3. HTML

    4. El Lenguaje de Marcado de Hipertexto (HTML) es el código que se utiliza para estructurar y desplegar una página web y sus contenidos. Por ejemplo, sus contenidos podrían ser párrafos, una lista con viñetas, o imágenes y tablas de datos.

  4. XHTML

    5. XHTML (lenguaje de marcado de hipertexto extensible) es una familia de lenguajes de marcado XML que reflejan o amplían versiones del ampliamente utilizado lenguaje de marcado de hipertexto (HTML).

  5. HTML5

    6. HTML5 (Hypertext Markup Language 5) es un lenguaje de marcado utilizado para estructurar y presentar documentos de hipertexto en la World Wide Web . Fue la quinta y última versión principal de HTML, que ahora es una recomendación retirada del World Wide Web Consortium (W3C). La especificación actual se conoce como HTML Living Standard.

III. Otros lenguajes relacionados al desarrollo web

  1. CSS/CSS3

    2. CSS es la versión básica y no es compatible con el diseño receptivo. CSS3, por otro lado, es la última versión y admite un diseño receptivo. CSS no se puede dividir en módulos, pero CSS3 se puede dividir en módulos .

  2. Javascript

    3. JavaScript es un lenguaje de programación que los desarrolladores utilizan para hacer páginas web interactivas. Desde actualizar fuentes de redes sociales a mostrar animaciones y mapas interactivos, las funciones de JavaScript pueden mejorar la experiencia del usuario de un sitio web.

  3. Ajax

    4. AJAX permite a las aplicaciones web validar información específica en formularios antes de que los usuarios los envíen. Por ejemplo, cuando un nuevo usuario crea una cuenta, la página web puede verificar automáticamente si hay un nombre de usuario disponible antes de que el usuario pase a la siguiente sección.

  4. Tecnologias y lenguajes del lado del servidor

    5. Los lenguajes del lado servidor son aquellos reconocidos, cargados e interpretados por el propio servidor y que se envían al cliente en un formato comprensible para él, de manera que puedan ser entendidos directamente por el navegador, puesto que se encontrarían traducidos en lenguaje del lado cliente.

  5. Navegadores

    6. Un navegador web es un programa que permite ver la información que contiene una página web. El navegador interpreta el código, HTML generalmente, en el que está escrita la página web y lo presenta en pantalla permitiendo al usuario interactuar con su contenido y navegar.

  6. Motores renderizados

    7. Un motor de renderizado es un componente de software básico de los principales navegadores web. La función principal de un motor de navegación es transformar los documentos HTML y otros recursos de una página web en una representación visual interactiva en el dispositivo del usuario.

IV. Herramientas de desarrollo

  1. Software WYSIWYG

    2. Este término se utiliza para describir un tipo de editor que permite a los usuarios ver en tiempo real cómo quedará su contenido final mientras lo están editando.

  2. Otras alternativas

    1. Figma

      2. Figma es un editor de gráficos vectorial y una herramienta de generación de prototipos, principalmente basada en la web, con características off-line adicionales habilitadas por aplicaciones de escritorio en macOS y Windows.

    2. InVisión

      3. InVision es una plataforma de diseño digital que se ha convertido en una herramienta esencial para profesionales de UX/UI. Permite la creación de wireframes, prototipos interactivos, y diseños de alta fidelidad, ofreciendo una amplia gama de funcionalidades como animaciones, transiciones y gestos.

    3. Zeplin

      4. Zeplin ayuda a los equipos a cumplir la promesa del diseño. Esta plataforma de entrega de diseño conecta el diseño con el desarrollo. Además, proporciona un espacio de trabajo organizado para publicar diseños donde todos los equipos de productos pueden colaborar y enviar estupendos productos juntos.

    4. GitHub

      5. GitHub es una plataforma donde puedes almacenar, compartir y trabajar junto con otros usuarios para escribir código. Almacenar tu código en un "repositorio" en GitHub te permite: Presentar o compartir el trabajo. Seguir y administrar los cambios en el código a lo largo del tiempo.


  3. Inspección de código desde el navegador

    4. La herramienta de inspección de código del navegador se encuentra integrada en cualquier navegador. Deberás utilizarla cuando con el resto de herramientas no puedas obtener la información deseada. Permite: Ver el código HTML de algún elemento de la web.

Tema 2: El diseño de una pagina web

  1. Tipos de paginas web

    2. Institucionales o de negocio: Estas páginas están diseñadas para representar una organización o empresa. Su objetivo principal es informar sobre la misión, visión, servicios, productos y ofrecer contacto con la entidad. Ejemplos son los sitios web de grandes corporaciones, bancos, ONGs y gobiernos.

    Personales o portafolios: Son páginas creadas por individuos para mostrar sus habilidades, proyectos o información personal. Son comunes entre profesionales creativos como diseñadores, escritores, fotógrafos, y freelancers, quienes utilizan estos sitios como una forma de promocionar su trabajo.

    Micrositios: Se trata de pequeñas páginas web, generalmente asociadas a una campaña específica o a un producto dentro de una web más grande. Su propósito es muy concreto y suelen tener una duración temporal. Son comunes en marketing o lanzamientos de productos.

    Blogs: Los blogs son sitios que se actualizan regularmente con entradas (posts), generalmente de manera cronológica. Pueden ser personales o comerciales y abarcar cualquier tema. Se utilizan para compartir información, opiniones, noticias o experiencias de una manera informal y cercana.

    Educativos: Este tipo de páginas están destinadas a ofrecer contenido de aprendizaje, recursos didácticos o plataformas para la enseñanza. Son comunes entre universidades, escuelas, academias en línea y programas de capacitación. También incluyen cursos online (e-learning).

    Comercio electrónico (E-commerce): Estas páginas están diseñadas para vender productos o servicios en línea. Integran carritos de compra, pasarelas de pago y sistemas de inventario. Ejemplos son Amazon, eBay, o sitios individuales de tiendas.

    Portales: Son sitios web que agrupan una gran cantidad de contenido diverso, sirviendo como puerta de entrada a varios servicios o secciones. Los portales suelen ofrecer noticias, foros, recursos, y enlaces a otros sitios web. Ejemplos incluyen Yahoo o MSN.

    Noticias o revistas: Estas páginas se especializan en publicar noticias actuales o artículos en formato de revista. Pueden ser de acceso gratuito o por suscripción, y cubren desde temas generales hasta sectores específicos, como deportes, política o tecnología. Ejemplos son sitios como BBC, El País, o Wired.

Tema 3: Estructura de una pagina web

  • I. Jerarquia de directorio de un proyecto Web

    • La estructura de directorios es una jerarquía organizada de carpetas y subcarpetas en un dispositivo de almacenamiento, como un disco duro o un servidor. Esta jerarquía facilita la ubicación, el acceso y la gestión de archivos almacenados en el sistema.

  • II. Elementos de Estructura para el diseño de una pagina Web: Concepto
    Los elementos de estructura en el diseño de una página web son los componentes fundamentales que organizan y dan forma al contenido de una página. Son esenciales para asegurar una presentación clara, coherente y navegable, tanto para los usuarios como para los motores de búsqueda.

    1. Regla utilizada para validar y certificar una pagina web

      2. Certificados SSL/TLS (HTTPS)

      Regla: El uso de un certificado SSL/TLS para encriptar la conexión entre el navegador y el servidor. Este protocolo asegura que los datos enviados entre el usuario y el sitio web estén protegidos.

      Validación: Las autoridades certificadoras (CA) emiten certificados SSL. Un sitio con un certificado válido muestra el prefijo https:// en la barra de direcciones, junto con un candado.

      Validación de código HTML y CSS

      Regla: Las páginas web deben seguir las especificaciones del W3C (World Wide Web Consortium) para HTML y CSS, garantizando que el código esté estructurado de manera adecuada y que sea accesible para todos los navegadores.

      Validación: Hay herramientas como el W3C Validator que permiten verificar si el código HTML y CSS de un sitio cumple con los estándares.


    2. Regla para escribir html

      3. Estructura básica Cada documento HTML debe seguir una estructura básica.

      Etiquetas deben estar correctamente anidadas Las etiquetas HTML deben cerrarse en el orden correcto, es decir, una etiqueta abierta dentro de otra debe cerrarse antes de cerrar la etiqueta externa.

      Cerrar siempre las etiquetas Aunque algunos navegadores permiten omitir el cierre de ciertas etiquetas (como ,
      , etc.), es una buena práctica cerrarlas siempre.


    3. Estructura basica de un archivo web

      4. HTML (HyperText Markup Language): Es el lenguaje de marcado utilizado para crear la estructura de una página web. Define los diferentes elementos como párrafos, encabezados, imágenes, enlaces, formularios, etc. Un archivo HTML básico.

      CSS (Cascading Style Sheets): Es el lenguaje utilizado para definir el estilo y la presentación de una página web. Un archivo CSS básico.

      JavaScript: Es el lenguaje de programación que añade interactividad y comportamiento a la página web.
